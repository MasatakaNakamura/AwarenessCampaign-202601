# 1on1管理システム 要件定義書

バージョン: 1.0
作成日: 2026-01-25
作成者: 開発チーム

## 1. 概要
- 目的: 組織内で上長と部下の1on1を効率的に管理・記録し、継続的な成長とエンゲージメント向上を支援するためのシステムを構築する。
- 対象範囲: 1on1のスケジューリング、議題管理、議事録・アクションアイテムの記録、フィードバックの集計・閲覧、レポート出力、権限管理。

## 2. 背景とゴール
- 課題: 手作業の記録、属人化した議題・フォローの管理、振り返りデータの未整備。
- ゴール:
  - 1on1実施の記録を標準化し検索・集計可能にする。
  - フォローアップやアクションの進捗を可視化する。
  - 必要なアクセス権限を確保しつつ操作を簡便にする。

## 3. ステークホルダー
- エンドユーザー: 社員（被面談者）、マネージャー（面談実施者）
- 管理者: 人事/HR部門
- システム管理者: DevOps/IT

## 4. 用語定義
- 1on1: 上長と部下の1対1ミーティング
- 議題(Topic): ミーティングで話す項目
- アクションアイテム(AI): 実施すべきタスク

## 5. 要求（機能要件）
5.1 認証・認可
- S1: 社員は社内SSO（例: SAML/OIDC、あるいはLDAP）でログインできること。
- S2: ロールは少なくとも「一般ユーザー（社員）」「マネージャー」「管理者」を持つこと。

5.2 1on1スケジューリング
- S3: ユーザーは1on1を予定として作成・編集・キャンセルできること。
- S4: 予定には日時、場所（オンラインリンク）、参加者（上長/部下）、タグを設定できること。
- S5: カレンダー同期（Google Calendar / Microsoft 365）はオプションで連携可能であること。

5.3 議題管理
- S6: 面談前に議題を作成・共有できること（タイトル、詳細、優先度、所要時間）。
- S7: 議題は面談当日まで編集可能で、履歴が残ること。

5.4 議事録・アクション
- S8: 面談中および面談後に議事録を作成・保存できること。
- S9: アクションアイテムを議事録に紐づけ、担当者・期限・状態（未着手/進行中/完了）を管理できること。
- S10: アクションのリマインダー通知（期限前）が可能であること。

5.5 フィードバック・評価
- S11: 面談ごとに簡易フィードバック項目（例: 満足度、コメント）を記録できること。

5.6 検索・フィルタ
- S12: 期間・参加者・タグ・ステータス等で履歴検索・フィルタができること。

5.7 レポート・ダッシュボード
- S13: マネージャー向け: 自チームの1on1実施状況・未対応アクション一覧を表示。
- S14: 人事向け: 組織全体の集計（1on1実施率、平均満足度、主要トピックの傾向）を出力できること。
- S15: CSV/Excelでのエクスポート機能。

5.8 通知
- S16: 予定作成/変更/リマインダー等をメールまたは社内チャット（Slack等）へ通知できること。

5.9 インポート/エクスポート
- S17: 既存の1on1データやCSVからの一括インポートをサポートすること。

## 6. 非機能要件
- N1: 可用性: 99.5%／月を目標とする。
- N2: セキュリティ: データはTLSで保護。機密情報は保存時に暗号化するポリシーを想定。
- N3: 認可: ロールに基づくアクセス制御（RBAC）を実装すること。
- N4: パフォーマンス: 100ms以下のページ応答（一般的な一覧表示）を目標とする。
- N5: ロギング: 操作ログ（作成・更新・削除）を保持し、監査が可能であること。
- N6: 保守性: APIドキュメント（OpenAPI）を備え、CI/CDで自動デプロイ可能であること。

## 7. データ設計（概略）
- User: id, name, email, role, department
- OneOnOne: id, organizer_id, participant_id, start_at, end_at, location, status, tags
- Topic: id, oneonone_id, title, description, priority, created_by, created_at
- Memo/Minutes: id, oneonone_id, content, created_by, created_at
- ActionItem: id, oneonone_id, title, assignee_id, due_date, status, created_at

## 8. 画面（主要画面）
- ログイン画面
- ダッシュボード（自分の次回1on1、未完了アクション）
- 予定一覧（検索・フィルタ）
- 予定詳細（議題、議事録、アクション）
- 議題作成・編集モーダル
- レポート画面（CSVエクスポート）
- 管理画面（ユーザー・ロール管理）

## 9. API 要件（概略）
- 認証: /auth/* (SSO連携)
- 予定: GET/POST/PUT/DELETE /api/oneonones
- トピック: GET/POST/PUT/DELETE /api/topics
- 議事録: GET/POST/PUT /api/minutes
- アクション: GET/POST/PUT/DELETE /api/actions
- レポート: GET /api/reports

## 10. 権限マトリクス（簡易）
- 一般ユーザー: 自分の予定の作成・編集、参加する1on1の閲覧・議題作成
- マネージャー: 自チームメンバーの1on1予定作成・閲覧・レポート閲覧
- 管理者: 全データの閲覧・エクスポート・ユーザー管理

## 11. 非機能制約・運用ルール
- 最低限のプライバシー対応（面談内容の閲覧ポリシーを明確化）。
- データ保持ポリシー（例: 議事録は7年保管可、または組織ルールに従う）。

## 12. 受入基準
- 全ての主要機能がE2Eで動作すること（認証、予定管理、議事録、アクション、通知）。
- CSVエクスポートおよび基本的なレポートが作成できること。
- 主要APIの自動テストが用意されていること。

## 13. マイルストーン（提案）
1. プロトタイプ（MVP）: スケジューリング、議題、議事録、アクション基本機能（4週間）
2. 通知・カレンダー連携追加（2週間）
3. レポート・管理者機能追加（2週間）
4. セキュリティ・運用整備（2週間）

## 14. 今後の検討事項
- 多言語対応、モバイル最適化、より詳細な権限階層、エンタープライズ向け監査機能。
