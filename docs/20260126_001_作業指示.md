# 1on1管理システム プロジェクト作成 — 作業指示書

参照: [docs/20260126_001.md](docs/20260126_001.md)

作業者: AI（このリポジトリ上で自動的に作業を行う前提）

作成日: 2026-01-25

目的:
- 本リポジトリに対して、新規 Spring Boot ベースの 1on1 管理システム プロジェクトを初期構成する。

前提条件:
- リポジトリはローカルにクローン済みで、作業ブランチが作成されている（例: feat/001-create-project）。
- Java 開発環境（JDK 11+）、Maven または Gradle が使用可能。

作業範囲:
- Spring Initializr でのプロジェクトスケルトン生成
- 依存関係追加（最低: Spring Web, Spring Data JPA, H2 または PostgreSQL ドライバ, Lombok など）
- 基本パッケージとエントリクラス、README の作成
- Dockerfile と docker-compose.yml の追加（必要に応じ）
- GitHub Actions による簡易 CI 設定
- ドキュメント（セットアップ手順）記載

作業手順（AI が実行する順序）:
1. Spring Initializr でプロジェクトを生成（MavenまたはGradleを選択）。
2. 生成されたスケルトンを `backend/` またはルートに配置し、`pom.xml`/`build.gradle` を確認。
3. 必須依存関係を `pom.xml`/`build.gradle` に追加。
4. パッケージ構成を作成: `com.example.oneonone`（例）下に `controller`, `service`, `repository`, `model`, `config` を作成。
5. アプリケーションのエントリポイント（`Application` クラス）とサンプルのヘルスチェック用コントローラを追加。
6. `README.md` にセットアップ手順、起動方法、環境変数、DB接続情報を記載。
7. `Dockerfile` と `docker-compose.yml` を追加（アプリ + DB サービス）。
8. GitHub Actions ワークフローを `.github/workflows/ci.yml` に追加（ビルド & 単体テスト実行）。
9. 変更をコミットし、リモートにプッシュ（初期コミット）。

成果物（リポジトリ内のパス）:
- プロジェクトソース: ルートまたは `backend/`
- 作業指示書: [docs/20260126_001_作業指示.md](docs/20260126_001_作業指示.md)
- README: `README.md`（ルート）
- Docker関連: `Dockerfile`, `docker-compose.yml`
- CI 設定: `.github/workflows/ci.yml`

検証手順:
- `mvn -f <path> clean package` または `./gradlew build` が成功すること。
- コンテナ起動: `docker-compose up --build` でアプリが起動し、`/actuator/health`（または作成したエンドポイント）が 200 を返すこと。

見積り（概算）:
- 初期スケルトン作成 + README: 1〜2時間
- Docker/Compose + CI 設定: 1時間
- 合計: 約 2〜3時間

注意事項・確認事項:
- 実装言語は Java（Spring Boot）を想定。別の言語を希望する場合は指示すること。
- DB は開発時は組み込み DB（H2）で進め、ステージ/本番は PostgreSQL を想定する構成にする。
