# テーマ：コンテキストエンジニアリング (Context Engineering)
Anthropic社の提唱する概念（https://www.anthropic.com/engineering/effective-context-engineering-for-ai-agents）をベースにします。
- **概念**: AIエージェントに与える情報（コンテキスト）を最適化し、信頼性と精度を高める技術。
- **なぜ重要か**: トークン数が増えるとAIの注意力（Attention）は下がる。「とりあえず全部RAGに突っ込む」ではAIは賢くならない。「情報を整理し、最小限かつ最適な形で渡す」設計が必要。
- **フック（魅力）**:
    - 「AIが嘘をつく（ハルシネーション）」は、AIのせいではなく、あなたの"渡し方（コンテキスト設計）"が悪いのかもしれない。
    - これは「AIへの指示出し」ではなく、「新しい時代の要件定義・設計力」である。
    - 現場で使える「ドキュメント整理」や「情報設計」のスキルそのものである。
