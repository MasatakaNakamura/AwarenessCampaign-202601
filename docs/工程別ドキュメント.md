## システム開発の工程別・主要成果物一覧

### 1. 要件定義工程 (Requirements Definition)

システムの「何を作るか」を明確にするフェーズです。

* **業務要件定義書:** ビジネス上の課題、解決策、新業務フローなど。
* **機能要件定義書:** 画面、帳票、バッチ処理など、システムが提供する機能。
* **非機能要件定義書:** 性能、信頼性、セキュリティ（特に重要）、運用保守要件。

### 2. 基本設計 / 外部設計工程 (External Design)

ユーザーから見える部分の仕様を決定します。

* **システム構成図:** ネットワーク、サーバー、クラウドサービス（AWS/Azure/GCP等）の構成。
* **画面一覧・画面遷移図:** ユーザー導線の定義。
* **画面レイアウト・帳票定義書:** UI/UXの詳細。
* **データモデル設計書 (ER図):** エンティティ間の関係性。
* **外部I/F設計書:** 外部システムやAPIとの連携仕様。

### 3. 詳細設計 / 内部設計工程 (Internal Design)

開発者がプログラムを実装するための設計図です。

* **クラス図・シーケンス図:** 処理のロジックやオブジェクト間のやり取り。
* **モジュール設計書:** 個別のプログラムやコンポーネントの処理内容。
* **テーブル定義書:** 各カラムの型、制約、インデックス設定。
* **APIリファレンス:** エンドポイント、リクエスト/レスポンス形式。

### 4. 実装 / テスト工程 (Testing)

作成したプログラムが正しく動くか検証します。

* **ソースコード:** プログラム本体。
* **単体テスト計画書・結果報告書:** メソッドや関数単位の検証。
* **結合テスト計画書・結果報告書:** モジュール間や画面間の連携検証。
* **システムテスト計画書・結果報告書:** 要件定義に基づいた全体動作の検証。
* **運用準備・移行計画書:** 本番切り替えの手順。

## 【まとめ】工程と資料の対応表

| 工程 | 主な成果物 |
| --- | --- |
| **要件定義** | 業務フロー、機能・非機能要件定義書、用語集 |
| **基本設計** | システム構成図、画面遷移図、ER図、API一覧 |
| **詳細設計** | クラス図、シーケンス図、テーブル定義、CRUD図 |
| **テスト** | テスト計画書、テストケース、バグ管理表、結果報告書 |
| **リリース・運用** | 移行手順書、運用マニュアル、保守手順書 |
